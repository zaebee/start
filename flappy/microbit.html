Flappy Micro:bit — лист заданий

Краткий набор миссий для урока по разбору Flappy Bird. Выбирайте уровень сложности и выполняйте — в конце подготовимся к финальной «битве».

Урок
Разбор Flappy Bird на Micro:bit
Время
80 минут

Миссии (выберите 1–2)

  • Уровень 1 — Закрепление (15–20 мин)

    Быстрые правки для понимания механики и синтаксиса.

    • 1. Уменьшите время паузы → игра станет быстрее.
    • 2. Поставьте птицу стартовать в Y = 1 или 3.
    • 3. Уберите кнопку B — пусть птица падает автоматически (имитация гравитации).

  • Уровень 2 — Игровые фичи (20–25 мин)

    Добавьте счёт, ускорение и визуальные подсказки.

    • 1. Счёт — +1 за каждое успешно пройденное препятствие.
    • 2. Игра постепенно ускоряется: уменьшайте паузу по таймеру.
    • 3. При смерти показывайте счёт (scroll или showNumber).

  • Уровень 3 — Креатив (25–30 мин)

    Добавьте одну заметную фишку, которую можно использовать в финале.

    • Бонусы: дополнительные очки за «звёздочку».
    • Множественные жизни: 3 жизни вместо одного удара.
    • Альтернативный контроль: акселерометр вместо кнопок.
    • Анимация птицы: меняйте яркость или мигание при полёте.

Подсказки

Подумайте, какая механика будет честной в соревновании: ускорение, бонусы или жизни — влияет на баланс между «лучшей» и «самой сложной» версиями.

Короткие сниппеты

Гравитация (JS):
basic.forever(function() {
    bird.change(LedSpriteProperty.Y, 1)
    basic.pause(700)
})
Увеличение скорости со временем (псевдо):
let speed = 1000
// в цикле: basic.pause(speed)
// каждые N тиков: speed = Math.max(150, speed - 50)
Счёт: увеличивать, когда препятствие уходит влево:
if (obstacle.get(LedSpriteProperty.X) == 0) {
  score += 1
}
// показывать score по кнопке B

Подготовка к финальной битве

  • Определите, какие фичи разрешены (жизни, бонусы, ускорение).
  • Каждая команда делает <= 2 заметных улучшения.
  • На следующей сессии перенесём логику в браузер и устроим турнир.

Быстрый чек-лист для ученика

  1. Запусти примерный код на Micro:bit (демо на экране).
  2. Внеси 1–2 изменения — протестируй.
  3. Если есть идея — реализуй фичу и протестируй снова.
  4. Запиши своё имя + краткое описание фичи (для финальной битвы).

Домашнее (необязательно)

Подумай над одной фичей, которую можно показать в финале: скетч интерфейса в браузере, идея бонуса, или простая баланс-механика.
